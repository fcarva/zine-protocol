# ESTAR Magazine - Contexto aplicado ao Zine Protocol

Data: 2026-03-02
Fonte principal: `https://estarmagazine.com`

## 1. Leitura de DNA visual

1. Navegacao minima: poucos links, caixa alta, ritmo seco.
2. Hierarquia curta: capa/objeto no centro, pouco texto por bloco.
3. Atmosfera de publicacao independente: linguagem direta e sem excesso de UI.
4. Uso de simbolos e divisores para ritmo editorial (ex.: separadores compactos).
5. Estrutura modular: home, catalogo, contato, espaco expositivo.

## 2. Traducao para o Zine Protocol

1. Home ganhou `context strip` com links curtos e separadores (`★`) para reforcar navegacao editorial.
2. Home ganhou bloco `Exhibition space` para encaixar narrativas curatoriais curtas + mini-galeria.
3. Mantido principio de imagem primeiro e copy curta nos cards.
4. Sem copiar identidade literal; adaptacao orientada ao produto (arquivo aberto + apoio).

## 3. Regras de adaptacao adotadas

1. Manter Flexoki como base cromatica.
2. Evitar excesso de botoes secundarios.
3. Limitar texto introdutorio por bloco.
4. Priorizar capa, autoria e contexto tecnico.
5. Preservar CTA principal: `Apoiar este zine`.

## 4. Proximos passos recomendados

1. Variacao de grid por secao (arquivo, series, curadoria) para reforcar ritmo editorial.
2. Modo "issue": agrupar zines em edicoes tematicas com data e curadoria.
3. Pagina de contato/editorial submissions mais enxuta, no mesmo tom minimalista.
4. Sequencia de leitura (proximo/anterior) no zine aberto com micro-navegacao discreta.
